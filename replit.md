# Книжный Искатель — Book Search RAG Service

## Overview

A full-stack web application for searching text fragments in books and answering questions based on book content (RAG pattern).

## Stack

- **Monorepo tool**: pnpm workspaces
- **Node.js version**: 24
- **Package manager**: pnpm
- **TypeScript version**: 5.9
- **API framework**: Express 5
- **Database**: PostgreSQL + Drizzle ORM
- **Validation**: Zod (`zod/v4`), `drizzle-zod`
- **API codegen**: Orval (from OpenAPI spec)
- **AI**: OpenAI via Replit AI Integrations (gpt-5.2 for Q&A)
- **Frontend**: React + Vite, TailwindCSS, shadcn/ui, Framer Motion

## Features

1. **Fragment Search**: Search for relevant text fragments from uploaded books — returns 3-5 best matching passages with source information (book, chapter, position).
2. **Q&A Mode**: Ask questions based on book content — AI generates an answer using found fragments and shows citations.
3. **Book Management**: Upload TXT files, add title/author metadata, delete books.
4. **Honest fallback**: If no relevant content is found, the system clearly reports this to the user.

## Backend Routes

- `GET /api/books` — list all uploaded books
- `POST /api/books` — upload a book (multipart/form-data: file, title?, author?)
- `DELETE /api/books/:id` — delete a book
- `POST /api/search/fragments` — search for text fragments `{ query, topK? }`
- `POST /api/search/answer` — get AI answer with citations `{ query, topK? }`

## Algorithm

Text is split into ~600-character overlapping chunks with chapter detection. Search uses TF-IDF-like cosine similarity scoring. AI answers are generated by gpt-5.2 using the top-K most relevant chunks as context.

## TypeScript & Composite Projects

- `lib/*` packages are composite and emit declarations via `tsc --build`.
- Root `tsconfig.json` lists lib packages as references.
- Run codegen: `pnpm --filter @workspace/api-spec run codegen`

## Root Scripts

- `pnpm run build` — runs `typecheck` first, then recursively runs `build` in all packages
- `pnpm run typecheck` — runs `tsc --build --emitDeclarationOnly` using project references
